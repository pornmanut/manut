# Coding Standards

## File Structure
- Astro components: `src/components/`
- Pages: `src/pages/`
- Layouts: `src/layouts/`
- Blog posts: `src/content/blog/YYYY/`
- Images: `public/images/`

## Code Style
- Use 2 spaces for indentation in Astro files
- Use 4 spaces for indentation in TypeScript files
- Use PascalCase for component names (Header.astro, PostCard.astro)
- Use TypeScript for all new files
- Keep components small and focused

## Styling
- Use CSS custom properties from BlogLayout.astro
- Don't use inline styles
- Maintain Medium-inspired design system
- Use Charter font for body text, Inter for UI

## Import Rules
- Import Astro components at top of file
- Use TypeScript interfaces for props
- Follow existing naming patterns
